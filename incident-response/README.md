# Incident Response

Incidents are normal. What separates good teams from great ones is how quickly they detect, coordinate, mitigate, and learn.

This section contains:

- [Severity Levels](severity-levels.md) — SEV1 through SEV4 definitions and examples.
- [Runbook Template](runbook-template.md) — a starting point for any production runbook.
- [Postmortem Template](postmortem-template.md) — a blameless learning template.

## Guiding Principles

- **Safety first.** Protect people, data, and customers before preserving velocity.
- **Coordinate out loud.** Use a single incident channel and a single incident commander.
- **Mitigate before root-causing.** Fix the bleeding, then investigate.
- **Learn without blame.** The goal of a postmortem is resilient systems, not accountability theater.
