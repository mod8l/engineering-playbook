# Hiring

I hire for judgment, learning velocity, and collaborative impact. Technical depth matters, but the best engineers I have worked with also know how to simplify, communicate, and raise the people around them.

This section contains:

- [Role Template](role-template.md) — a starting point for Staff/Principal ML Engineer job descriptions.
- [Interview Loop](interview-loop.md) — a four-session loop with scorecards and anti-patterns.
- [Take-Home Exercise](take-home-exercise.md) — a bounded ML system design prompt and rubric.

Every hiring process leaks signal. My goal is to make the leaks intentional: candidates should understand what we value, interviewers should calibrate on the same dimensions, and debriefs should focus on evidence rather than gut feel.
